# Справочник форматов источников (PDF, Excel, DOCX)

ИИ и скрипты могут извлекать вопросы из разных видов файлов. Ниже — **типовые шаблоны**, как устроены блоки; реальный файл может отличаться — тогда нужна эвристика (номер вопроса, буквы вариантов, строка «Правильный ответ»).

## 1. Excel / Google Sheets

### Вариант A: одна строка — один вопрос

| Колонка | Смысл |
|--------|--------|
| A | № |
| B | Текст вопроса |
| C–F | Варианты A–D |
| G | Правильный ответ (`A`, `B`, `1`, или полный текст варианта) |
| H | Предмет / категория (для маппинга на `subjectSlug`) |
| I | Тема (для `topicNameRu`) |

### Вариант B: варианты в одной ячейке

Одна строка: вопрос | `Вар1;;;Вар2;;;Вар3;;;Вар4` | правильный индекс `1–4` | предмет.

Разделитель `;;;` или `|` — задай явно в промпте, если известен.

### Вариант C: экспорт из тестовых систем

Часто встречаются колонки: `Question`, `Option1`, `Option2`, `Option3`, `Option4`, `Correct`, `Subject`, `Topic`, `Difficulty`.

---

## 2. PDF

Типичные паттерны в тексте после извлечения (OCR или копирование):

```
1. Текст вопроса до знака вопроса?
A) первый вариант
B) второй вариант
C) третий вариант
D) четвёртый вариант
Правильный ответ: B
```

Или:

```
Вопрос 12. ...
а) ...
б) ...
в) ...
г) ...
Ответ: в)
```

ИИ должен:

- Нормализовать префиксы вариантов (`A)`, `а)`, `1.`).
- Сопоставить строку «Правильный ответ» с одним из вариантов.
- Сохранить формулы как LaTeX в `contentRu`: `$...$` или `\(...\)`.

---

## 3. Word (DOCX)

Часто используются нумерованные списки для вопросов и буквенные подсписки для ответов. После конвертации в текст правила те же, что для PDF. Таблицы Word — аналог Excel: строка = вопрос.

---

## 4. Изображения / скан

Нужен OCR. После распознавания применяются те же правила, что для PDF; добавь в промпт: «исправь типичные OCR-ошибки в кириллице и в индексах A/B/C/D».

---

## 5. Рекомендуемый промпт-уточнение под конкретный файл

Добавь к основному промпту один абзац:

> Источник: [Excel один лист / PDF с нумерацией / …].  
> Разделитель вариантов в ячейке: [например `;;;`].  
> Колонка предмета: [буква или название].  
> Кодировка правильного ответа: [буква / номер / полный текст].  
> Если встречается «несколько верных» — используй `type: "multiple_choice"`.

---

## 6. Маппинг предмет → `subjectSlug`

Слаг **всегда** берётся из ответа `GET /bulk/catalog`, а не придумывается. Примеры соответствий (зависят от вашей базы):

| Текст в файле | Действие |
|---------------|----------|
| «Математическая грамотность» | найти subject с похожим `name.ru` |
| «Физика профиль» | то же |
| Нет колонки предмета | спросить пользователя или положить в тему `Импорт без предмета` и один общий `subjectSlug` по умолчанию |

---

## 7. Ограничения API (напоминание)

- Batch: до **50** вопросов за запрос.
- Варианты ответа: до **12** на вопрос.
- Эндпоинты `/bulk/*` (кроме `GET /bulk/status`) доступны только при `BULK_IMPORT_ENABLED=true` на сервере.
